# Changelog

本项目遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [1.1.2] - 2026-06-17

### 修复
- 修复"同一行内有时无法添加样式标签"的不稳定问题:此前选区获取依赖 `document.activeElement` 的 shadow DOM 链路,某些时机会解析到错误的根而取不到真实选区,导致 `execCommand` 作用在错误目标、什么都没插入。现改为穿透 shadow DOM 扫描所有可编辑区、定位真正持有选区的那个;并在 `execCommand` 返回失败时用 DOM `extractContents` 手动包裹兜底。

### 新增
- `debug` 配置项:开启后样式应用失败会把原因写入日志,并在编辑器内弹出 toast 提示,方便排查。

## [1.1.1] - 2026-06-16

### 修复
- 修复样式按钮"只生效一次/部分按钮从不生效"的问题:此前包裹依赖 Anki 内置 `window.wrap`,其依赖的 `focusedInput` 在字段内容变化后会被清空,导致后续点击静默失效。现改为在 mousedown 时捕获选区,点击时重新聚焦字段并用 `document.execCommand("insertHTML")` 直接对该选区操作,不再依赖 Anki 焦点状态。
- 修复"更多"下拉的关闭监听器随每次重渲染累积的泄漏(改为全局只绑定一次)。

## [1.1.0] - 2026-06-16

### 新增
- **拖拽排序**:直接在编辑器工具栏按住按钮左右拖动即可调整顺序;「更多 ▾」下拉内的项可上下拖动排序;下拉打开时还可在工具栏与下拉之间互相拖动。松手后立即生效并自动写回配置(per-class `order`),重启后保持。

## [1.0.0] - 2026-06-16

首个版本（V1.0 MVP + 部分 V1.1/V1.2 体验项）。

### 新增
- CSS class 自动解析：基础/逗号/嵌套/复合选择器、伪类、`@media`、中文 class，自动忽略 `@keyframes`/`@font-face`/`:root` 变量，去重并以最后定义为准。
- 编辑器工具栏动态样式按钮，随笔记类型切换自动刷新；无可用 class 时不显示任何元素。
- 选中文本包裹：有/无选区处理、空标签光标定位、同 class Toggle 取消、不同 class 嵌套叠加。
- 内置忽略列表（`card`、`cloze`、`nightMode` 等）。
- JSON 配置：全局设置 + 按笔记类型的 per-class 配置（显示名、排序、标签、快捷键、标记颜色、可见性）。
- 按钮样式预览（安全属性）、Tooltip CSS 摘要、快捷键绑定。
- 按钮溢出处理：`dropdown` / `newline` / `scroll`。
- 中英文国际化，跟随 Anki 语言。
- `Tools → Style Buttons 设置…` 菜单入口。

### 异常处理
- CSS 解析失败、配置损坏、JS 执行异常均 graceful 降级，不影响编辑器正常使用，异常写入日志。
