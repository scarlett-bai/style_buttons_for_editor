# Anki 卡片编辑器 CSS 样式快捷插入插件 — 产品需求文档（PRD）

---

## 一、产品概述

### 1.1 产品名称

**Style Buttons for Editor**（暂定）

### 1.2 一句话描述

在 Anki 卡片编辑器的字段输入框上方，根据当前笔记类型（Note Type）模板中定义的 CSS 样式，自动生成对应的快捷按钮；用户选中文本后点击按钮，即可将所选文本自动包裹在对应的 HTML 标签中，无需手写代码。

### 1.3 要解决的问题

| 痛点 | 说明 |
|------|------|
| **手写 HTML 繁琐** | 用户在编辑器中需要手动输入 `<span class="bold">...</span>` 等代码，效率低下 |
| **记忆负担重** | 用户自定义了多种 CSS class，但编辑时很难记住所有可用的 class 名称 |
| **容易出错** | 手写标签容易拼写错误、漏写闭合标签，导致卡片样式渲染异常 |
| **工作流割裂** | 用户不得不频繁切换到模板编辑器查看有哪些可用的 class |

### 1.4 目标用户

- 使用自定义 CSS 样式美化卡片的 Anki 用户
- 有多套笔记模板、每套模板包含不同 CSS class 的重度用户
- 希望提升编辑效率、减少手动输入 HTML 的用户

---

## 二、核心概念与术语

| 术语 | 解释 |
|------|------|
| **Note Type（笔记类型）** | Anki 中的模板，包含字段定义、卡片模板（正面/背面 HTML）和共享样式（CSS） |
| **Styling（模板 CSS）** | 每个 Note Type 中 "Styling" 区域定义的 CSS 代码 |
| **CSS Class** | 在 Styling 中定义的 `.className { ... }` 规则 |
| **编辑器工具栏** | Anki 编辑器中字段输入框上方的按钮栏（加粗、斜体、下划线等按钮所在的区域） |
| **样式按钮** | 本插件新增的按钮，每个按钮对应一个 CSS class |

### 3.2 F1 — CSS Class 自动解析

#### 3.2.1 功能描述

插件在编辑器打开时，自动读取当前笔记所属 **Note Type** 的 **Styling（CSS）** 内容，从中解析出所有定义的 CSS class。

#### 3.2.2 解析规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 基础 class | 提取形如 `.className { ... }` 的定义 | `.bold { font-weight: bold; }` → class 名：`bold` |
| 多 class 逗号分隔 | 一条规则中有多个 class 时分别提取 | `.red, .blue { ... }` → 提取 `red` 和 `blue` |
| 嵌套/组合选择器中的 class | 形如 `.card .highlight` 的选择器，提取最后一个 class | `.card .highlight { ... }` → 提取 `highlight` |
| 忽略 Anki 内置 class | 自动忽略 `.card`、`.cloze`、`.nightMode` 等 Anki 内置/系统级 class | `.card { ... }` → 不提取 |
| 忽略伪类/伪元素 | 忽略 `:hover`、`::before` 等 | `.tip:hover { ... }` → 提取 `tip`，忽略伪类部分 |
| 去重 | 同一个 class 在 CSS 中出现多次只保留一个 | — |

#### 3.2.3 内置忽略列表（默认）

以下 class 默认不生成按钮（用户可在配置中调整）：

```
card, card1, card2, card3, cloze, nightMode, night_mode, 
replay-button, typepad, typeBad, typeGood, typeMissed
```

#### 3.2.4 刷新时机

- 打开编辑器时
- 切换笔记类型（Note Type）时
- 从模板编辑器返回时（如果用户修改了 CSS）

### 3.3 F2 — 样式按钮动态渲染

#### 3.3.1 按钮位置

在编辑器工具栏的现有按钮（加粗、斜体等）**之后**，增加一个**视觉分隔符**，然后显示本插件的样式按钮组。

```
┌─────────────────────────────────────────────────────────────────┐
│ [B] [I] [U] [···] │ [bold] [highlight] [red] [vocab] [grammar] │
│   Anki 原生按钮     │          本插件生成的样式按钮                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 3.3.2 按钮外观

| 属性 | 规则 |
|------|------|
| **按钮文字** | 默认显示 class 名称（如 `bold`）；用户可在配置中自定义显示名称 |
| **按钮样式预览** | 按钮本身尽可能应用对应 CSS 样式的关键属性来预览效果（如：`bold` 按钮的文字加粗显示，`red` 按钮文字显示为红色），让用户直观感知 |
| **Tooltip** | 鼠标悬停时显示：class 名称 + 快捷键（如有） + CSS 属性摘要 |
| **按钮尺寸** | 与 Anki 原生工具栏按钮保持一致的高度和内边距 |

#### 3.3.3 按钮过多时的处理

当 CSS class 数量较多（超过一定阈值，如 8 个），提供以下方案（可在配置中选择）：

- **方案 A（默认）**：工具栏中显示前 N 个按钮 + 一个 **「更多 ▼」** 下拉菜单，点击后展开完整列表
- **方案 B**：单独新增一行工具栏
- **方案 C**：全部显示，工具栏可横向滚动

#### 3.3.4 无可用 class 时的表现

如果当前笔记类型的 CSS 中没有解析到任何自定义 class，则：
- 不显示任何样式按钮
- 不显示分隔符
- 编辑器外观与未安装插件时完全一致

### 3.4 F3 — 选中文本包裹

#### 3.4.1 基本操作流程

```
用户操作流程：

1. 在编辑器的某个字段中输入或已有文本
2. 用鼠标或键盘选中一段文本（如选中 "重要概念"）
3. 点击工具栏上的样式按钮（如 [highlight]）
4. 选中文本被替换为：<span class="highlight">重要概念</span>
5. 编辑器中的显示效果保持不变（仍显示纯文本或带格式文本）
6. 在复习/预览卡片时，文本会应用对应的 CSS 样式
```

#### 3.4.2 详细行为规则

| 场景 | 行为 |
|------|------|
| **有选中文本** | 将选中文本包裹在 `<span class="xxx">选中文本</span>` 中 |
| **无选中文本** | 在光标位置插入 `<span class="xxx"></span>`，并将光标定位到 span 标签内部，方便用户直接输入 |
| **选中文本已有相同 class 包裹** | **移除**该 class 的包裹（Toggle 行为），即点击同一按钮可以取消样式 |
| **选中文本已有不同 class 包裹** | 在外层再嵌套一层新 class 的包裹（支持多样式叠加） |
| **选中内容包含 HTML 标签** | 保持内部 HTML 结构不变，仅在最外层包裹 |
| **跨字段选择** | 不处理（与 Anki 原生行为一致，编辑器不支持跨字段选择操作） |

#### 3.4.3 包裹标签类型

默认使用 `<span>` 标签。用户可在配置中为特定 class 指定不同的标签：

| 配置项 | 说明 | 示例 |
|------|------|------|
| `span`（默认） | 行内元素 | `<span class="bold">文本</span>` |
| `div` | 块级元素 | `<div class="card-section">文本</div>` |
| 自定义标签 | 任意 HTML 标签 | `<mark class="highlight">文本</mark>` |

#### 3.4.4 快捷键支持

- 每个样式按钮可绑定一个快捷键
- 默认不分配快捷键（避免与 Anki 原生快捷键冲突）
- 用户可在配置中自行设置，如 `Ctrl+Shift+1` 对应第一个样式按钮

### 3.5 F4 — 插件配置

#### 3.5.1 配置入口

- **方式 1**：Anki 菜单栏 → 工具 (Tools) → Style Buttons Settings
- **方式 2**：插件管理器 → 选中本插件 → Config（Anki 原生配置方式）

#### 3.5.2 配置界面

提供一个 GUI 配置窗口（非纯 JSON 编辑），包含以下配置区域：

---

**配置区域 1：全局设置**

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `enabled` | Boolean | `true` | 插件开关 |
| `default_tag` | String | `"span"` | 默认包裹标签 |
| `button_overflow_mode` | Enum | `"dropdown"` | 按钮过多时的处理方式：`dropdown` / `newline` / `scroll` |
| `button_overflow_threshold` | Integer | `8` | 触发溢出处理的按钮数量阈值 |
| `show_css_preview_tooltip` | Boolean | `true` | 是否在 tooltip 中显示 CSS 属性预览 |
| `ignored_classes` | List[String] | （见 3.2.3） | 全局忽略的 class 列表 |

---

**配置区域 2：按笔记类型的按钮配置**

这是配置的核心区域。左侧显示笔记类型列表，右侧显示该类型下解析出的 class 列表及其配置。

每个 class 的可配置项：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `visible` | Boolean | `true` | 是否显示为按钮 |
| `display_name` | String | （class 名称） | 按钮上显示的文字 |
| `order` | Integer | （解析顺序） | 按钮排列顺序（数字越小越靠前） |
| `tag` | String | `""` | 该 class 使用的包裹标签（空字符串表示使用全局默认） |
| `shortcut` | String | `""` | 快捷键绑定（如 `"Ctrl+Shift+1"`） |
| `icon` | String | `""` | 自定义图标路径（可选，高级功能） |
| `color` | String | `""` | 按钮标记颜色（用于快速视觉区分） |

## 四、非功能需求

### 4.1 兼容性

| 项目 | 要求 |
|------|------|
| **Anki 版本** | 支持 Anki 2.1.55+（Qt6 优先），兼容 Anki 23.x / 24.x |
| **操作系统** | Windows / macOS / Linux |
| **编辑器场景** | 支持以下所有编辑场景：添加笔记（Add）、编辑笔记（Edit）、浏览器中编辑（Browser Editor） |
| **与其他插件共存** | 不应与常见热门插件产生冲突（如 Edit Field During Review、Frozen Fields 等） |

### 4.2 性能

| 指标 | 要求 |
|------|------|
| CSS 解析耗时 | 单个 Note Type 的 CSS 解析应在 **50ms** 以内完成 |
| 按钮渲染耗时 | 按钮组渲染不应造成编辑器打开时的可感知延迟（**< 100ms**） |
| 文本包裹操作 | 点击按钮后文本替换应在 **50ms** 以内完成 |
| 内存占用 | 插件额外内存占用不超过 **10MB** |

### 4.3 稳定性

- 插件在任何情况下不应导致 Anki 崩溃
- CSS 解析失败时（如 CSS 语法异常），应 graceful 降级：不显示按钮，不影响编辑器正常使用
- 所有异常应写入 Anki 的 debug 日志

### 4.4 国际化（i18n）

- 插件界面支持 **中文** 和 **英文**
- 根据 Anki 的语言设置自动切换
- 按钮上的 class 名称不翻译（保持 CSS 原始名称或用户自定义名称）

## 五、技术要点提示

> 以下内容供开发者参考，非严格限制。

### 5.1 关键 API / Hook

| 用途 | 建议使用的 Anki API |
|------|---------------------|
| 向编辑器工具栏添加按钮 | `editor.setupButtons` / `setupEditorButtons` hook |
| 获取当前笔记类型的 CSS | `note.note_type()["css"]` 或 `model["css"]` |
| 在编辑器中操作选中文本 | 通过 `editor.web.eval()` 执行 JavaScript，操作编辑区域的 DOM / `document.execCommand` 或 Anki 的 `setFormat` 等 |
| 监听笔记类型切换 | `editor.currentField` 变化 / `editor.setNote` hook |
| 配置管理 | `mw.addonManager.getConfig()` / `writeConfig()` |

### 5.2 CSS 解析建议

- 建议使用正则表达式或轻量级 CSS parser（如 `tinycss2`、`cssutils`）来解析 CSS
- 需要处理 CSS 注释 `/* ... */` 的情况
- 需要处理 media query 嵌套的情况
- 需要处理 CSS 变量定义 `:root { --var: ... }` 的情况（应忽略）

### 5.3 编辑器架构注意事项

- Anki 2.1.50+ 使用 Svelte + TypeScript 重写了编辑器前端
- 工具栏按钮的添加可能需要同时处理 Python 端（hook 注册）和 JS 端（DOM 操作）
- 文本包裹操作在 JS 端执行，需要与编辑器的 undo/redo 机制兼容

## 六、用户使用场景（User Stories）

### 场景 1：基本使用

> 作为一名日语学习者，我在笔记模板 CSS 中定义了 `.vocab { color: blue; font-weight: bold; }` 和 `.grammar { color: green; font-style: italic; }` 两个样式。
> 我希望在添加卡片时，选中一个单词后点击工具栏上的 `vocab` 按钮，该单词就自动被 `<span class="vocab">` 包裹。这样我复习时能看到蓝色加粗的单词效果。

### 场景 2：切换笔记类型后按钮自动更新

> 我有 "英语词汇" 和 "医学知识" 两个笔记类型，它们的 CSS 样式完全不同。
> 当我在添加卡片窗口从 "英语词汇" 切换到 "医学知识" 时，工具栏上的样式按钮应该自动更新为 "医学知识" 模板中定义的 class。

### 场景 3：取消已应用的样式

> 我给一段文字添加了 `highlight` 样式后发现不合适，我希望选中这段文字再次点击 `highlight` 按钮，就能移除包裹的 `<span class="highlight">` 标签。

### 场景 4：使用快捷键快速操作

> 我最常用的三个样式是 `bold`、`highlight`、`red`，我在配置中为它们分别绑定了 `Ctrl+Shift+1`、`Ctrl+Shift+2`、`Ctrl+Shift+3`。
> 编辑卡片时我选中文本后直接按快捷键就能应用样式，无需移动鼠标。

### 场景 5：隐藏不需要的按钮

> 我的模板 CSS 中定义了很多用于卡片布局的 class（如 `.container`、`.flex-row`），这些 class 不是用于标记文本内容的，我不希望它们出现在工具栏上。
> 我在配置中将这些 class 的 `visible` 设为 `false`，工具栏只保留我真正需要的样式按钮。

### 场景 6：空白模板

> 我新建了一个笔记类型，还没有添加任何自定义 CSS。
> 此时编辑器工具栏上不应出现任何本插件的按钮，编辑器看起来和没装插件一样。

## 七、交互流程图

### 7.1 编辑器打开时的初始化流程

```
编辑器打开
    │
    ▼
读取当前笔记的 Note Type
    │
    ▼
获取 Note Type 的 CSS (Styling)
    │
    ▼
解析 CSS，提取所有 class
    │
    ▼
过滤：移除忽略列表中的 class
    │
    ▼
读取插件配置：获取各 class 的配置（visible / order / display_name 等）
    │
    ▼
有可显示的 class？
    │
    ├── 否 ──→ 不添加任何按钮，流程结束
    │
    └── 是 ──→ 按 order 排序
                  │
                  ▼
              按钮数量 > 溢出阈值？
                  │
                  ├── 否 ──→ 全部渲染为工具栏按钮
                  │
                  └── 是 ──→ 前 N 个渲染为按钮 + "更多" 下拉菜单
                              │
                              ▼
                          渲染完成，注册快捷键，流程结束
```

### 7.2 点击样式按钮的处理流程

```
用户点击样式按钮（class = "xxx"）
    │
    ▼
检测编辑器中是否有选中文本
    │
    ├── 无选中文本
    │       │
    │       ▼
    │   在光标位置插入 <span class="xxx"></span>
    │   将光标移动到 span 内部
    │       │
    │       ▼
    │   流程结束
    │
    └── 有选中文本
            │
            ▼

        检测选中文本是否已被 <span class="xxx"> 包裹
            │
            ├── 是（已有相同 class）
            │       │
            │       ▼
            │   移除该 <span class="xxx"> 标签，保留内部文本内容
            │       │
            │       ▼
            │   流程结束（Toggle 取消样式）
            │
            └── 否（未被相同 class 包裹）
                    │
                    ▼
                获取该 class 的配置（tag 类型）
                    │
                    ▼
                将选中文本包裹为 <{tag} class="xxx">选中文本</{tag}>
                    │
                    ▼
                保持包裹后的文本为选中状态
                    │
                    ▼
                流程结束
```

### 7.3 笔记类型切换流程

```
用户在编辑器中切换 Note Type
    │
    ▼
触发 Note Type 变更事件
    │
    ▼
清除当前工具栏中所有本插件生成的样式按钮
    │
    ▼
重新执行初始化流程（同 7.1）
    │
    ▼
新的样式按钮渲染完成
```

## 八、边界情况与异常处理

### 8.1 边界情况

| 编号 | 场景 | 预期处理方式 |
|------|------|-------------|
| E1 | CSS 中没有任何自定义 class | 不显示任何按钮，不显示分隔符 |
| E2 | CSS 语法错误导致解析失败 | 记录日志，不显示按钮，不影响编辑器正常功能 |
| E3 | class 名称包含特殊字符（如中文、连字符、下划线） | 正常支持。CSS 中合法的 class 名称均应正确处理，如 `.my-class`、`.my_class`、`.日本語` |
| E4 | CSS 中有 `@media`、`@keyframes` 等 at-rule | 解析 `@media` 内部的 class；忽略 `@keyframes`、`@font-face` 等不含 class 的规则 |
| E5 | CSS 中有 `!important` | 不影响解析，正常提取 class 名称 |
| E6 | 选中文本中包含图片、音频等媒体标签 | 正常在最外层包裹，不修改内部媒体标签 |
| E7 | 选中文本跨越多个已有 HTML 标签 | 在选区最外层包裹。如选中 `<b>粗体</b>普通`，包裹为 `<span class="xxx"><b>粗体</b>普通</span>` |
| E8 | 快捷键与 Anki 原生或其他插件冲突 | 在设置快捷键时进行冲突检测并提示用户，不自动覆盖 |
| E9 | 用户在模板编辑器中修改了 CSS 后返回编辑器 | 重新解析 CSS，更新按钮（可能需要通过 hook 监听模板保存事件） |
| E10 | 同一个 class 在 CSS 中被定义多次（覆盖写法） | 去重后只生成一个按钮；Tooltip 中显示最后一次定义的属性 |
| E11 | CSS 使用了 `@import` 导入外部样式 | 不支持解析外部导入，仅解析 Styling 文本框中的内容，在文档中注明此限制 |
| E12 | 编辑器中使用了 Markdown 或其他标记语法的插件 | 本插件只操作 HTML 层面，与 Markdown 插件的兼容性需测试 |

### 8.2 异常处理策略

| 异常类型 | 处理方式 |
|----------|---------|
| CSS 解析异常 | 捕获异常，写入 debug 日志，按钮区域不渲染，编辑器正常工作 |
| 配置文件损坏/格式错误 | 使用默认配置，弹出一次性提示告知用户配置已重置 |
| JS 端执行异常（文本包裹操作失败） | 捕获异常，显示简短错误提示（如编辑器底部的 toast 通知），不修改字段内容 |
| Hook 注册失败（可能因 Anki 版本不兼容） | 记录日志，提示用户当前 Anki 版本可能不支持，插件功能禁用 |
